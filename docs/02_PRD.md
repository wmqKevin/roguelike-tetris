## 1. 背景与目标

### 1.1 背景
- CEO 一句话目标：**做一个炫酷的肉鸽俄罗斯方块网页小游戏**。
- 俄罗斯方块认知门槛极低、几乎人人会玩，但纯俄罗斯方块缺乏长期留存动力。
- 肉鸽（单局随机 + 局内成长 + 死亡重来）能注入"再来一局"的成瘾循环，在保留低门槛的同时提供重玩深度。
- "炫酷"是差异化关键：同类产品（Tetris Effect 偏音游向、Tetris 99 偏竞技向）较少在"局内强化构筑 × 视觉冲击"上发力，是本项目切入点。

### 1.2 产品目标
- **北极星指标**：单局平均时长 3–8 分钟；首日"死亡后立刻再来一局"的比例 ≥ 50%。
- **MVP 目标**：跑通"方块下落消除 → 攒能量 → 局内强化三选一 → 强化改变方块玩法 → 撑过递增难度 → 死亡重来"最小闭环，验证玩法有趣。
- **非目标（MVP 不做）**：联机/对战、排行榜、账号系统、付费、皮肤、任务/成就系统、Meta 永久养成（均显式延后，见 §4）。

### 1.3 成功标准
- 玩家在 5 分钟内能理解核心循环并完成至少 1 次强化选择。
- 单局无卡死、无崩溃、帧率达标（见 §5）。
- QA 依据 §8 验收标准全部通过，方可发布。

## 3. 核心玩法（俄罗斯方块 × 肉鸽融合）

### 3.1 核心循环（一句话）
> 方块下落消除 → 累积能量 → 触发局内强化三选一 → 强化直接改变方块玩法 → 撑过递增难度 → 死亡 → 重来构筑新流派。

### 3.2 融合点（关键：强化如何与方块玩法耦合，非简单叠加）
强化（Relic / Buff）直接作用于方块系统的参数，而非独立数值。四类耦合：

1. **改变方块本身**：如"重型块"（每 N 个方块出现一个 2×2 不可旋转重块）、"幽灵块"（半透明下落、落地才显形）、"分裂块"（落地分裂成两半）。
2. **改变下落 / 消除规则**：如"重力反转"片段、"斜消"（斜线连通可消）、"连锁消除"（消除后补位下落触发连锁）。
3. **改变消除效果**：如"爆炸"清除消除点周围格、"激光"清整行 / 整列、"冰冻"保留消行供下次组合。
4. **资源与节奏耦合**：消除产生能量，能量满触发强化抉择；连消 / Tetris（一次消 4 行）产能量翻倍，鼓励高难度操作。

> 与游戏策划 ZI-58 的边界：本 PRD 定义融合规则与下限（强化 ≥ 10 种、特殊方块 ≥ 2 种），具体效果与数值以策划案为准；冲突交 Leader 裁决。

### 3.3 单局结构
- 单局 = 多个"波次 / 阶段"，每阶段难度（下落速度 + 特殊块频率）递增。
- 每完成一个阶段 或 能量满 → 强化抉择（三选一）。
- 死亡条件：方块堆到顶端（经典 Top-out）。
- 单局无"胜利"终点，目标为存活时长 / 分数 / 阶段数（极限挑战）。

### 3.4 操作（经典 + 必要扩展）
- 左右移动、软降（加速下落）、硬降（瞬间落底）、顺 / 逆时针旋转、Hold（暂存方块）。
- 桌面默认键位：← → 移动、↓ 软降、空格硬降、↑ / X 顺时针、Z 逆时针、C / Shift 暂存。
- 移动端（P1）：触屏滑动 + 按钮区。

## 5. 非功能需求

- **性能**：桌面目标 60 FPS；移动端 ≥ 30 FPS；单局内存占用 < 150 MB；首屏加载 < 3 秒（4G 模拟）。
- **兼容性**：Chrome / Edge / Firefox / Safari 近 2 年版本；桌面优先，移动端 Safari / Chrome 兼容（P1）；分辨率适配 1280×720 ~ 1920×1080，移动竖屏自适应（P1）。
- **包体**：首屏资源 < 5 MB（gzip 后）；无后端依赖；纯静态托管可部署（GitHub Pages）。
- **可访问性**：关键操作有键盘等价；颜色对比度满足 WCAG AA（P1）；提供"减少动效"开关（P1）。
- **安全 / 隐私**：无账号、无网络上报；本地存档不含敏感信息。
- **稳定性**：单局无卡死、无死循环、无空指针崩溃；连续游玩 30 分钟内存增长 < 10%。

## 6. 关键用户流程

1. **首次进入**：加载 → 标题页 → [开始游戏] → 单局开始 → 首屏操作引导（3 步）→ 自由游玩。
2. **单局内**：方块下落 → 操作 → 消行得分 + 攒能量 → 能量满 → 暂停弹强化三选一 → 选择 → 强化生效 → 继续 → 难度递增 → 循环。
3. **死亡**：Top-out → Game Over 页（分数 / 阶段 / 本局强化）→ [再来一局] / [返回标题]。
4. **暂停**：任意时刻 [Esc / P] → 暂停面板（继续 / 重开 / 设置 / 返回标题）。

## 7. 关键页面 / 交互低保真说明

- **标题页**：游戏 Logo + [开始游戏] / [设置]；背景动态粒子（炫酷定调）。
- **游戏主界面**：
  - 中央：10×20 棋盘（标准 Tetris 比例）。
  - 左侧：Hold 框 + 本局已获得强化图标列表。
  - 右侧：Next 预览（3 个）+ 分数 / 阶段 / 能量条。
  - 顶部：当前阶段标识。
- **强化抉择面板**：暂停游戏，中央弹出 3 张卡片（图标 + 名称 + 描述 + 稀有度），键盘 1/2/3 或点击选择。
- **Game Over 页**：大分数 + 阶段数 + 本局强化回顾（横向列表）+ [再来一局] 主按钮。
- **暂停面板**：半透明遮罩 + 继续 / 重开 / 设置 / 返回标题。
- **设置页（P1）**：音量滑条 / 键位重绑 / 震动开关 / 减少动效开关。

## 8. 验收标准（可量化，QA 据此测试）

> 每个功能点对应可测条件；数值类阈值以策划案 `03_Balance.md` 为最终配置，本表给定性 / 边界。

- **F01 方块核心**：7 种 Tetromino 均能出现且旋转遵循选定旋转系统（建议 SRS）规则；硬降瞬间落底；Hold 可暂存并交换，同一方块不可连续 Hold 两次。**验收**：测试用例覆盖 7 × 4 旋转态无越界 / 穿墙。
- **F02 消行计分**：单 / 双 / 三 / Tetris 计分正确（基础分示例：100 / 300 / 500 / 800，可被强化修饰）；combo 连消计数正确。**验收**：脚本化输入序列，分数与期望误差 = 0。
- **F03 Next / Hold**：Next 队列展示 ≥ 3 个且与内部队列一致；Hold 初始为空。**验收**：渲染队列 == 内部队列。
- **F04 难度递增**：每消除 N 行（建议 10）提升一档速度；速度随行数单调递增。**验收**：记录下落间隔随行数单调递减。
- **F05 死亡判定**：新生成方块无法放置（与已有块重叠或超出顶端）即 Game Over。**验收**：构造堆满场景触发 Top-out，无异常。
- **F06 能量与抉择**：能量满自动触发抉择面板并暂停；三选一互不重复；选择后强化立即生效。**验收**：能量阈值固定，抉择池去重。
- **F07 强化池**：MVP ≥ 10 种强化，每种有明确参数变更（非纯文案）；同局可叠加。**验收**：逐项验证参数生效（如"爆炸消除"半径 = 1 格）。
- **F08 特殊方块**：≥ 2 种，出现频率可配置。**验收**：在指定阶段必出现且行为正确。
- **F09 阶段系统**：≥ 3 阶段，阶段切换有视觉 / 数值反馈。**验收**：阶段数与难度参数同步递增。
- **F10 重开**：Game Over 后 [再来一局] 在 ≤ 1 秒内进入新局，状态完全重置（无残留强化 / 分数）。**验收**：重开后内部状态 == 初始状态。
- **F11 视觉反馈**：消行触发闪光 + 粒子 + 屏幕震动；Tetris 反馈强度 > 单行。**验收**：震动幅度可量化（位移像素），强度递增正确。
- **F12 最高分存档**：刷新页面后最高分保留；清浏览器数据后归零。**验收**：localStorage 读写正确。
- **F13 暂停**：暂停时游戏循环停止（方块不下落、不计时 / 计分）；恢复后无跳帧。**验收**：暂停 10 秒后分数 / 状态无变化。
- **F14 触屏（P1）**：左 / 右滑动移动、下滑硬降、点击旋转、长按 Hold。**验收**：触屏操作与键盘等价。
- **F15a 音效（P0）**：消行 / 强化 / 硬降触发至少有音效反馈。**验收**：P0 事件触发音效无遗漏，可被设置静音。
- **F15b BGM（P1）**：BGM 可循环可静音。**验收**：音乐通道与音效通道独立控制。
- **F16–F19（P1）**：见对应功能，验收标准在迭代启动时细化。
- **非功能验收**：桌面 60 FPS（Chrome DevTools 实测，10 分钟平均 ≥ 58）；移动 ≥ 30 FPS；首屏 < 3 秒；连续 30 分钟内存增长 < 10%。

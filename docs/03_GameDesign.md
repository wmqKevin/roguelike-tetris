# 03_GameDesign - 肉鸽俄罗斯方块玩法设计

## 1. 设计目标

项目定位：纯前端单机 Web 小游戏，将经典俄罗斯方块的即时操作、空间规划与肉鸽的随机强化、阶段挑战、死亡重来结合。

MVP 目标：
- 保留标准 10x20 俄罗斯方块核心体验。
- 增加最小肉鸽循环：阶段推进、随机强化、随机词缀、失败重开。
- 所有强化必须影响落块、消行、棋盘或阶段压力，避免做成独立加分系统。

## 2. 核心玩法

### 2.1 棋盘与方块

- 棋盘：10 列 x 20 可见行，顶部额外 2 行隐藏生成区。
- 方块：使用标准 7 种 Tetromino：I、O、T、S、Z、J、L。
- 生成规则：默认 7-bag 随机，每袋包含 7 种方块各 1 个，抽空后重新洗袋。
- 预览：默认显示接下来 3 个方块。
- Hold：默认可持有 1 个方块，每个下落方块只能 Hold 1 次。
- 旋转：建议采用 SRS 旋转系统；MVP 可先实现基本踢墙。

### 2.2 基础操作

| 操作 | 键位建议 | 说明 |
|---|---:|---|
| 左移 / 右移 | ← / → 或 A / D | 水平移动 1 格 |
| 软降 | ↓ 或 S | 加速下落，获得少量能量 |
| 硬降 | Space | 立即落地，按下落距离获得能量 |
| 顺时针旋转 | ↑ / X / K | 旋转 90 度 |
| 逆时针旋转 | Z / J | 旋转 -90 度 |
| Hold | C / Shift | 暂存或交换当前方块 |
| 使用主动技能 | 1 / 2 / 3 | 消耗能量或冷却 |
| 暂停 | Esc / P | 暂停游戏 |

### 2.3 胜利与失败

失败条件：
- 新方块生成位置被占用，无法进入棋盘。
- Boss 阶段倒计时结束且目标未完成。

阶段胜利条件：
- 普通阶段：完成指定消行数或目标分。
- 事件阶段：完成特殊条件，如清理指定垃圾行、达成连击、保护底线。
- Boss 阶段：在压力条满前累计造成足够伤害。

单局胜利条件：
- 完成第 5 章 Boss 阶段。

## 3. 核心循环

### 3.1 单阶段循环

1. 进入阶段，生成目标与词缀。
2. 玩家正常落块、消行、获得能量和碎片。
3. 消行、连击、T-Spin、特殊块触发构筑效果。
4. 完成目标后进入奖励选择。
5. 从 3 个随机强化中选择 1 个，必要时进入商店。
6. 难度提高，进入下一阶段。

### 3.3 v0.6 反馈层

- 竖屏 HUD 顶部目标拆为两层：主目标只显示「目标：再消 X 行」；副目标在「攒满能量」与「落 X 块」之间轮播，避免压住棋盘。
- 奖励 toast 在 compact / portrait 下固定为视口宽度减 28px、居中显示，并启用文字换行。
- 首个奖励选择后必须在 0.8-1.2 秒内播放演示反馈：
  - 精准硬降：能量飞字从棋盘飞入能量条，能量条脉冲。
  - 稳定预判：Next 首位闪烁高亮，并显示短飞字。
  - 技能奖励：全屏短闪、技能就绪飞字、按钮区域脉冲，并补贴一次试用能量。
- 音效使用 Web Audio 合成，不引入外部授权资源：硬降低频落地、1/2/3/4 行消除分层音、奖励亮音、技能短闪音、Game Over 下坠音；BGM 为本地合成循环音型，独立 music bus。
- 结算页在失败原因后追加「下局建议」，根据中路高堆叠、右侧井口被堵、连续无消行硬降、未拿稳定预判等模式给出短建议；本局新增图鉴与最高 Stage 徽章使用高亮色。

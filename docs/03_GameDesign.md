# 03_GameDesign - 肉鸽俄罗斯方块玩法设计

## 1. 设计目标

项目定位：纯前端单机 Web 小游戏，将经典俄罗斯方块的即时操作、空间规划与肉鸽的随机强化、阶段挑战、死亡重来结合。

MVP 目标：
- 保留标准 10x20 俄罗斯方块核心体验。
- 增加最小肉鸽循环：阶段推进、随机强化、随机词缀、失败重开。
- 所有强化必须影响落块、消行、棋盘或阶段压力，避免做成独立加分系统。

## 2. 核心玩法

### 2.1 棋盘与方块

- 棋盘：10 列 x 20 可见行，顶部额外 2 行隐藏生成区。
- 方块：使用标准 7 种 Tetromino：I、O、T、S、Z、J、L。
- 生成规则：默认 7-bag 随机，每袋包含 7 种方块各 1 个，抽空后重新洗袋。
- 预览：默认显示接下来 3 个方块。
- Hold：默认可持有 1 个方块，每个下落方块只能 Hold 1 次。
- 旋转：建议采用 SRS 旋转系统；MVP 可先实现基本踢墙。

### 2.2 基础操作

| 操作 | 键位建议 | 说明 |
|---|---:|---|
| 左移 / 右移 | ← / → 或 A / D | 水平移动 1 格 |
| 软降 | ↓ 或 S | 加速下落，获得少量能量 |
| 硬降 | Space | 立即落地，按下落距离获得能量 |
| 顺时针旋转 | ↑ / X / K | 旋转 90 度 |
| 逆时针旋转 | Z / J | 旋转 -90 度 |
| Hold | Shift / V | 暂存或交换当前方块 |
| 使用主动技能 | C / F / G | 消耗能量或冷却；行清除器默认用 C 释放 |
| 奖励选择 | 1 / 2 / 3 | 三选一强化选择 |
| 暂停 | Esc / P | 暂停游戏 |

### 2.3 胜利与失败

失败条件：
- 新方块生成位置被占用，无法进入棋盘。
- Boss 阶段倒计时结束且目标未完成。

阶段胜利条件：
- 普通阶段：完成指定消行数或目标分。
- 事件阶段：完成特殊条件，如清理指定垃圾行、达成连击、保护底线。
- Boss 阶段：在压力条满前累计造成足够伤害。

单局胜利条件：
- 完成第 5 章 Boss 阶段。

## 3. 核心循环

### 3.1 单阶段循环

1. 进入阶段，生成目标与词缀。
2. 玩家正常落块、消行、获得能量和碎片。
3. 消行、连击、T-Spin、特殊块触发构筑效果。
4. 完成目标后进入奖励选择。
5. 从 3 个随机强化中选择 1 个，必要时进入商店。
6. 难度提高，进入下一阶段。

### 3.2 单局循环

### 3.3 v0.6 反馈层

- 竖屏 HUD 顶部目标拆为两层：主目标只显示「目标：再消 X 行」；副目标在「攒满能量」与「落 X 块」之间轮播，避免压住棋盘。
- 奖励 toast 在 compact / portrait 下固定为视口宽度减 28px、居中显示，并启用文字换行。
- 首个奖励选择后必须在 0.8-1.2 秒内播放演示反馈：
  - 精准硬降：能量飞字从棋盘飞入能量条，能量条脉冲。
  - 稳定预判：Next 首位闪烁高亮，并显示短飞字。
  - 技能奖励：全屏短闪、技能就绪飞字、按钮区域脉冲，并补贴一次试用能量。
- 音效使用 Web Audio 合成，不引入外部授权资源：硬降低频落地、1/2/3/4 行消除分层音、奖励亮音、技能短闪音、Game Over 下坠音；BGM 为本地合成循环音型，独立 music bus。
- 结算页在失败原因后追加「下局建议」，根据中路高堆叠、右侧井口被堵、连续无消行硬降、未拿稳定预判等模式给出短建议；本局新增图鉴与最高 Stage 徽章使用高亮色。

### 3.4 v0.7 首奖保护与构筑引导

- 首个奖励选择后进入 1.5 秒安全演示窗口：暂停自动下落与玩家操作输入，完整播放奖励生效样例，再恢复危险状态。
- 首奖演示按奖励类型展示：
  - 精准硬降：送高落差 I 块、能量飞字、能量条脉冲。
  - 稳定预判：Next 首位闪烁，并强制下一袋出现 I 块用于理解预览收益。
  - 技能奖励：补贴一次技能能量，显示技能就绪飞字和释放提示。
- Stage 1-2 使用新手缓冲：有效重力间隔提高 25%，给玩家更多时间完成前两次强化选择。
- 首奖前 12 块内，如果堆叠进入可见第 6 行以上，自动触发一次顶部危险救场：清理最低一行，并显示「顶部危险」提示，让玩家理解这是缓冲机制。
- UPGRADES 区域显示「当前流派」与「推荐下一奖」，根据已持有强化标签引导硬降充能、预判、四消、技能或防守方向。
- 结算页在操作建议外追加「下局构筑」建议，把失败复盘转成下一局构筑目标。
- 关键反馈分层：首奖、Tetris 四消、技能释放分别使用不同全屏脉冲、粒子和飞字，避免所有高光事件使用同一种反馈。

### 3.5 v0.8 奖励试用、技能闭环与流派路线

- 520x390 横屏奖励阶段把危险信息收束为右侧单行状态提示，奖励弹窗出现时暂停顶部 toast 滚动；奖励卡高度压缩为两列布局。
- 首奖安全演示结束后追加 10 秒试用目标：精准硬降、稳定预判、行清除器等首奖会给出明确目标，首次成功使用后奖励 +20 能量 / +120 分并播放完成反馈。
- 主动技能显示三态反馈：
  - 可释放：技能行和能量状态高亮，行清除器在底行显示目标预览。
  - 释放成功：飞字与 toast 显示「C 释放最低行清除」或对应技能结果。
  - 不可释放：toast 明确显示「能量不足 100」或「当前不能释放」。
  - 输入：C/F/G 释放主动技能，1/2/3 只用于奖励选择，Hold 改为 Shift/V。
- 奖励卡展示推荐、流派核心、补短板、立即生效、解锁技能等标签；稀有/史诗卡使用更强边框与背景脉冲区分。
- 结算页「本局流派」显示路线进度，例如「清场流 2/3」，并保留下局构筑建议，把复盘转成下一局首要目标。

### 3.6 v0.9 技能爽感、首奖安全试用与长期目标前置

- 行清除器成功释放时使用约 1 秒中心飞字「行清除器发动」、短暂停顿式震屏、全屏闪光、粒子环和底行横向扫光；扣能后能量条进入高亮反馈，不足能量提示靠近技能/棋盘底部区域。
- 首奖若选择技能卡，进入「按 C 释放前危险暂停」安全试用窗口：自动下落与危险推进保持冻结，但 C 技能允许立即释放；完成释放后关闭保护并发放试用奖励。
- 首奖试用完成奖励从分数提示改为更明确的成长条文案：`+20 能量 / 徽章进度 +1`，让试用目标和长期成长建立连接。
- 390px 竖屏奖励界面改为聚焦读卡：默认聚焦推荐卡并显示完整说明，另外两张以可点选小卡片展示标题、稀有度和核心标签，避免三张正文同时挤占竖屏空间。
- 开局与首奖后在 HUD 前置「本局推荐流派 / 可解锁徽章 / 图鉴进度」和「你正在走：清场流」路线条；推荐、流派核心、补短板、立即生效、解锁技能沿用统一标签颜色体系。

### 3.7 v0.10 C 技能峰值重编排与飞字队列

- C / 行清除器释放成功后拆成三段：0-200ms 短暂停顿与震屏，200-900ms 中心大字「行清除器发动」并同步底行横向扫光，900-1400ms 显示 `-100` 扣能飘字。
- `skillFeedback`、`trialFeedback`、技能峰值等飞字通过场景队列错峰播放，默认保持约 380ms 间隔，避免释放瞬间多层文案同帧叠字。
- 首奖试用完成时底部目标区替换为 2 秒奖励条「试用完成 +20 能量 / +120 分 / 徽章进度 +1」，能量条绿色回填闪动；奖励条结束后显示「再消 N 行拿下一奖」。
- 能量不足或不可释放时，提示移动到 C 技能行附近，技能行红黄闪动并抖动 250ms，同时保留短 toast，失败原因不再被底部目标或棋盘内容淹没。
